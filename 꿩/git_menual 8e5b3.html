1. git-local(로컬 저장소)

  1. git-scm 설치
  1. 내 컴퓨터에 깃으로 관리할 폴더 생성
  1. 생성한 폴더로 들어간후 우클릭하여 git bash 열기
    • cli 입력창 띄우기 두가지 방법
      1. vscode
      1. git bash

3. 깃폴더 초기화 $ git init

  1. 파일생성$ touch sample.txt
  1. 깃 상태확인 $git status
  1. 깃에 파일 추가 $ git add sample.txt

    git add .git

  1. 아까와 다른 메시지 확인됨 $ git status
  1. 커밋 메시지와 함께 실행 $ git commit -m "sample txt created"
  1. 8번과정실행시 사용자 정보 입력하라는 메시지 나올경우 사용자 정보 등록한다
    • 저장소별 정보등록

      $ git config user.name "qwerewqwerew"

      $ git config user.email "qwerew0@gmail.com"

    1. $ git config --global user.email 깃허브에 등록한 이메일주소
    1. $ git config --global user.name 깃허브 사용자이름

2. git-hub(원격저장소)

  1. 깃허브회원가입
  1. 레파지토리 생성

3. 로컬과 원격연결

  1. 깃허브 레파지토리의 명령어 복사 git remote add origin 깃허브 레파지토리 주소
  1. 원격서버에 로컬파일 업로드 git push origin main
  • 오류1 :error: src refspec main does not match any

    $ git branch -m master main 브랜치명 변경

  • 오류2 : remote: Permission to qwerewqwerew/test.git

    $ git remote set-url origin https://토큰값@github.com/깃레파지토리주소

  1. 깃허브 계정에 sample.txt 확인

4. 로컬저장소 파일변경

  1. sample.txt 를 수정한다
  1. 스테이지에 올리기 git add . (작업폴더의 모든 변경사항 올린다)
  1. 커밋 실행 git commit -m "김망고 메인수정"
  1. git push origin [master:로컬저장소의 브랜치명] 으로 올린다
  • 에러: fatal: The current branch main has no upstream branch.

    원격저장소 명을 작성안했을때 뜨는 오류git

    git remote -v 로 원격저장소 이름을 찾은후

    git push 원격저장소명 브랜치명 으로 실행

  1. 깃허브에 확인

    히스토리를 클릭하여 원하는 지점으로 되돌릴수 있다.

5. 원격저장소의 변경내역 가져오기

  1. 원격저장소 파일수정
  1. 파일변경 → 커밋메시지입력 → 커밋버튼 클릭
  1. 원격저장소와 로컬저장소의 파일 내용이 서로 다르다
  1. 로컬의 파일을 변경후 저장
  1. add, commit, push 해본다
  1. 원격파일 받아오기 git pull origin main
  1. 로컬의 sample.txt 열어본다
  1. sample.txt 파일 수정
  1. add,commit,push 실행

7. 기타 깃 명령어

a. 깃 로컬 삭제

$ rm -rf .git

b. 커밋 히스토리 확인

$ git log

c. 사용자 정보확인

$ git config —list

d. 브랜치명 변경

$ git branch -m [OLD_BRANCH] [NEW_BRANCH]

e. 원격저장소 연결/끊기

확인 $ git remote -v

제거 $ git remote remove origin

연결 $ git remote add origin 레파지토리주소

f. 원격저장소 복제

git clone 레파지토리주소
